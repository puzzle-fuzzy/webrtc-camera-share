# Product Brief

## 定位

这是一个轻量的一发多收摄像头共享工具。发送端负责采集摄像头并为每个接收端建立独立 WebRTC 连接；Rust 后台只负责页面、房间状态和信令转发，不转发视频媒体。

## 核心流程

### 发送端

1. 生成高熵房间 ID 与访问码，或填写 6–32 位 ASCII 字母/数字。
2. 获取摄像头权限、加载公开 STUN 配置并连接信令服务。
3. 为每个加入的接收端创建 Offer 和独立 PeerConnection。
4. 展示在线及已连接接收端数量。
5. 连接暂时中断时尝试 ICE restart，超过恢复时限后释放失效 PeerConnection。
6. 停止、页面卸载或启动失败时取消配置请求，并释放摄像头、WebSocket、定时器和全部 PeerConnection。

### 接收端

1. 从分享链接读取房间信息，或手动填写。
2. 加载公开 STUN 配置，连接信令服务并完成首帧鉴权。
3. 鉴权成功后接收房间容量与临时 TURN 凭据。
4. 处理发送端 Offer，返回 Answer 并播放远端视频。
5. 发送端离线后保留房间连接并继续等待；失效 PeerConnection 超时后自动清理。

## 产品边界

- 每个房间最多一个发送端，接收端容量可配置为一至八个，默认八个。
- 不包含账号、持久化房间、录制、聊天、音频或媒体服务器。
- 默认提供公共 STUN；生产环境可配置自有 STUN，并在房间鉴权成功后下发 coturn REST API 临时凭据。
- 媒体仍是有界 P2P fan-out。更大规模需要引入 SFU，不能只提高房间人数上限。
- 房间状态仅保存在后台内存，后台重启后清空。

## UI 原则

- 使用 shadcn/ui `base-nova` 原生组件，不修改其颜色、字体、圆角和组件样式。
- 根节点固定使用 shadcn/ui 原生黑色主题，不提供主题切换。
- 自定义 class 仅用于页面排列、间距、响应式布局与视频宽高。
- 会话设置使用 `FieldGroup` 和 `Field`，状态使用 `Alert`，角色及人数使用 `Badge`。
- 发送和接收页面共享相同的信息结构与操作顺序。

## 安全与隐私

- 访问码只允许 6 到 32 位 ASCII 字母或数字，默认生成 128 位随机值；默认房间 ID 具有 96 位随机熵，后台以 SHA-256 摘要匹配访问码。
- 分享凭据写入 URL Fragment，正常 HTTP 请求不会携带 Fragment。
- WebSocket 请求 URL 只包含角色与房间 ID；访问码在连接建立后通过首个鉴权帧发送。
- 信令消息经过角色、类型、peer ID 与字段校验后才转发。
- 鉴权失败按来源 IP 与房间计数，超过阈值后临时封禁；状态表有容量和过期清理边界。
- 后台使用同时限制消息数量和总字节数的发送队列、心跳和空闲超时，并限制连接数、房间数、消息大小与文本/二进制发送速率。
- TURN 共享密钥不会下发；临时凭据只在房间鉴权成功后通过对应 WebSocket 返回，并按来源 IP 限制为每 10 分钟 64 个。
- `/metrics` 暴露房间、连接、排队信令字节、限流、队列过载和鉴权拦截计数，便于观察容量异常。
- HTTP 响应包含 CSP、Permissions Policy、Referrer Policy、Nosniff 与防嵌入策略。
- 后台日志不记录访问码和完整查询参数。

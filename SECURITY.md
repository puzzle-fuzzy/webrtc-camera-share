# Security Policy

## 支持范围

安全修复面向最新发布版本和 `main` 分支。旧版本请先升级到最新版本后再确认问题是否仍存在。

## 私密报告漏洞

请使用仓库的 GitHub Security Advisory 私密报告功能联系维护者，不要为未修复漏洞创建公开 Issue，也不要在公开日志中粘贴访问码、接收链接、TURN 密钥、指标 Token、服务器地址或其他凭据。

报告请至少包含：受影响版本或提交、复现条件、实际影响、最小复现步骤，以及你已采取的风险控制。维护者目标是在 5 个工作日内确认收到报告；修复和披露时间会根据影响范围协商。

## 安全测试边界

仅对你拥有或已获明确授权的系统进行测试。请避免影响真实用户的流量、房间、TURN 配额和基础设施。仓库内的集成测试、模拟摄像头 E2E 与回环地址冒烟是首选复现环境。

## 部署责任

生产部署必须使用 HTTPS、随机且独立的 TURN shared secret、受保护的 `/metrics`、受限的防火墙和可信反向代理。示例配置不能替代组织自身的密钥管理、监控、补丁和事件响应流程。
